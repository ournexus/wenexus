# 执行步骤

## 1. 收集当前状态

回顾本次会话的完整上下文，提取以下信息：

- **正在做什么**：当前任务的简短描述
- **做到哪了**：已完成的具体步骤
- **下一步**：明确的后续行动项
- **阻塞/注意事项**：遇到的问题、关键发现、需要记住的上下文（如果有）
- **关键文件**：本次涉及的重要文件路径（如果有）

## 2. 读取现有 Progress.md

读取项目根目录的 `progress.md`，理解其现有结构和内容。

## 3. 更新 Progress.md

在 `progress.md` 中重写内容，遵循以下原则：

- **保留长期信息**：架构决策、环境配置、CI/CD 状态等不变
- **更新「已完成」列表**：将本次完成的事项标记为已完成
- **更新「未完成」列表**：移除已完成的、添加新发现的待办
- **更新「当前状态」**：反映最新的项目状态
- **追加「上次会话快照」section**（在文件末尾、`---` 分隔线之前）：

```markdown
## 上次会话快照

> 更新时间：YYYY-MM-DD HH:MM

**正在进行**：[简短描述当前任务]

**本次完成**：
- [具体完成的事项 1]
- [具体完成的事项 2]

**下一步**：
- [ ] [具体行动项 1]
- [ ] [具体行动项 2]

**备注**：[阻塞点/关键发现/需要记住的上下文，没有则省略此行]
```

## 4. 确认

告知用户 Progress.md 已更新，简要说明更新了哪些内容。

# 注意事项

- 保持简洁，每条不超过一行
- 用具体的描述而非泛泛的总结（"实现了 auth middleware" 而非 "做了一些后端工作"）
- 如果用户通过 $ARGUMENTS 提供了额外备注，一并写入
- 如果「上次会话快照」section 已存在，覆盖它（只保留最新一次）
