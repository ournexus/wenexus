# WeNexus MVP 开发需求文档（AI 执行版）

**版本**: 0.1  
**范围**: Core Platform MVP（面向中国市场）  
**目标读者**: AI 开发代理 + 人类审阅者  

---

## 0. 文档使用方式（给 AI）

- 本文档是**需求与验收标准**，不是技术实现方案。
- 任何实现细节（框架、数据库、API 结构、状态管理）由 AI 自行拆解，但必须满足本文档的**功能与体验约束**。
- UI 必须严格遵循「理性国际主义 (RaialIeral)」：克制、网格、黑白灰为主、精密排版、无装饰。

---

## 1. 产品一句话定义

WeNexus 是一个 **多 Agent 自由会话的共识构建平台**：围绕“主题”，用户可选择/编排多个 AI Agent，进行群聊式讨论，获得结构化观点与阶段性共识输出，从而提升认知并形成可行动的沟通建议。

---

## 2. MVP 目标与非目标

### 2.1 MVP 目标（必须达成）

- 用户可以：
  - 浏览主题（topic）列表并搜索
  - 新建主题
  - 选择多个 Agent 加入会话
  - 与多个 Agent 自由对话（群聊）
  - 在会话中获得：
    - 结构化的观点分歧与共识要点
    - 可复用的“沟通策略/话术建议”（轻量版）
- 体验上：
  - 画面必须“建筑级”克制
  - 交互必须干脆、短促、可信赖

### 2.2 MVP 非目标（明确不做/可延期）

- 3D 星空漫游视图（共识星图）作为 V1.0/后续特性，MVP 可用 2D 关系/列表替代
- 价值观测试全流程与复杂画像系统（可留入口，不强制实现完整题库）
- 完整社区体系（权威等级、复杂互动、治理系统）
- 企业版、开放创建专家平台、第三方 API

---

## 3. 核心对象（概念层，不限定存储实现）

- **主题 Topic**
  - 用户讨论的中心对象，包含标题、简介、标签/领域、热度、争议度、更新时间
- **Agent**
  - 预置角色（5-10 个），每个 Agent 有：名称、角色简介、方法论、对话风格、擅长维度（逻辑/事实/价值观/共情/实用性）
- **会话 Conversation**
  - 绑定某个 Topic，包含多方参与（用户 + 多 Agent），支持连续对话
- **消息 Message**
  - 用户/Agent 的发言
- **共识状态 Consensus State（MVP 轻量）**
  - 用于展示当前阶段的：
    - 共识要点（已达成）
    - 分歧要点（未解决）
    - 下一步关键问题（推进讨论）

---

## 4. 用户旅程（MVP 必须闭环）

### 4.1 旅程 A：从浏览到进入主题会话

- 用户打开首页（探索）
- 浏览瀑布流/列表中的主题卡片
- 点击主题卡片进入主题详情/会话页
- 系统展示：
  - 主题摘要
  - 推荐 Agent（可编辑）
  - 历史会话或新会话入口

### 4.2 旅程 B：新建主题并发起多 Agent 会话

- 用户点击“新建主题”
- 输入：标题、简要背景（可选）、期望讨论目标（可选）
- 选择 Agent（至少 2 个，最多 N 个，N 由产品配置）
- 创建并进入会话

### 4.3 旅程 C：会话中获得结构化输出

- 用户自由提问/补充背景
- 多 Agent 轮流或协作回应（可由系统编排）
- 系统持续产出“结构化面板”：
  - 观点要点
  - 分歧点
  - 共识进度（MVP 用文本+百分比即可）
  - 下一步建议问题
- 用户可一键生成“沟通策略/话术建议”（轻量版）

---

## 5. 功能需求（按模块）

### 5.1 探索首页（Topic Feed）

#### 必须能力

- 支持主题列表展示（瀑布流或等宽栅格均可）
- 支持搜索：按主题标题/关键词
- 支持基础筛选（最少 1 个维度）：
  - 领域/标签 或 热度/时效

#### 卡片信息（必须完整呈现）

- 标题
- 争议度（数值或等级）
- 参与 Agent 数量
- 最新观点摘要（1-2 行）
- 共识收敛度（百分比 + 简短文字）

#### 验收标准

- 列表滚动 60fps（或接近），无明显卡顿
- 卡片信息层级清晰：用户 3 秒内可理解卡片在表达什么

### 5.2 主题详情（Topic Detail）

#### 必须能力

- 展示主题基本信息（标题、背景/摘要、标签）
- 展示当前会话入口（新会话/继续会话）
- 展示已选 Agent 列表，并允许编辑：
  - 添加/移除 Agent
  - 调整 Agent 顺序（可选）

#### 验收标准

- 进入详情页后，用户能在 1 次视线扫视内找到“开始讨论/继续讨论”入口

### 5.3 多 Agent 群聊会话（核心）

#### 必须能力

- 参与者：用户 + 多 Agent
- 消息形态：文本为主
- Agent 行为要求（MVP 最小集合）：
  - 每个 Agent 必须保持稳定的角色风格（方法论一致）
  - 在同一轮讨论中，Agent 的观点要尽量避免重复
  - 对用户输入不足时，应提出澄清问题，而非强行结论

#### 会话编排（不限定实现，但必须满足体验）

- AI 必须保证“可读性”：
  - 不出现同时 5 个 Agent 长篇输出淹没用户
  - 单条输出优先结构化（标题 + 要点），避免散文

#### 验收标准

- 任意一个会话回合，用户能清晰分辨“谁在说话、说的是什么立场/依据”
- 至少能产出 1 次“结构化共识面板”更新（见 5.4）

### 5.4 结构化面板：观点/分歧/共识（MVP 轻量）

#### 必须能力

- 在会话旁（或顶部/底部）常驻一个“结构化面板”，包含：
  - **共识要点**（已达成）
  - **核心分歧**（待解决）
  - **证据/依据缺口**（还需要什么信息）
  - **下一步关键问题**（推动讨论）
- 共识收敛度：以百分比表达（允许是启发式值），并配一句解释

#### 验收标准

- 面板内容必须可复制
- 面板更新频率不能过高（避免闪烁/噪音），建议“每 N 轮/用户触发”更新

### 5.5 沟通策略/话术建议（MVP 轻量）

#### 必须能力

- 用户在会话中可点击“生成沟通策略”
- 输出必须包含：
  - 目标对象画像（基于用户输入的合理推断，明确不确定性）
  - 沟通目标（共识点/底线/优先级）
  - 建议话术（至少 3 条，语气克制、符合中文真实表达）
  - 风险提示（可能踩雷点）

#### 验收标准

- 输出必须可复制
- 输出语言必须面向中国用户，避免翻译腔

## 6. Agent 体系（MVP 建议配置）

MVP 建议提供 5-10 个预置 Agent。PRD 示例：

- 苏格拉底（提问、追问假设）
- 爱因斯坦（第一性原理、类比解释）
- 法学专家（权利义务、规则边界）
- 经济学家（成本收益、激励机制）
- 心理学家（动机、情绪、认知偏差）
- 菜场大妈（经验主义、现实可行性）
- 职场老司机（组织博弈、执行路径）

### 6.1 Agent 输出格式约束（必须）

- 优先使用：
  - 1 行结论
  - 3-5 条要点
  - 必要时附“我还需要你补充 X 才能更准确”
- 禁止：
  - 长篇大论无结构
  - 无证据的武断结论

## 7. 设计与交互规范（理性国际主义 / 数字化建筑美学）

> 目标：**建筑级克制**。让内容成为主角，让界面像现代主义美术馆。

### 7.1 反设计原则（硬性禁止）

- 禁止渐变文字
- 禁止具象隐喻图标（握手/拼图/锁头等“廉价素材感”）
- 禁止装饰性边框/花纹
- 禁止“大圆角 + 弥散阴影 + 炫光输入框”的通用科技风

### 7.2 画布与层级

- 背景色：`#000000`（纯黑，禁止带蓝调的“科技黑”）
- 表面层：`#121212`（仅用于区分层级）
- 分割线：`rgba(255, 255, 255, 0.08)`，**1px**
- 质感：数字玻璃
  - 大量使用 `backdrop-filter: blur(40px)`
  - 透明度必须克制，避免浑浊“毛玻璃”

### 7.3 排版（Typography is Interface）

- 字体：
  - 英文/数字：Inter
  - 中文：思源黑体（Source Han Sans / Noto Sans SC）
  - 数据/元信息：等宽字体（如 JetBrains Mono / Roboto Mono）
- 字重策略：
  - Display（大标题/关键数据）：极细或极粗（二选一，保持系统一致）
  - Body：Regular 400
- 微排版：
  - 标题字间距略收紧
  - 小标签可全大写（英文）+ 宽字间距

### 7.4 色彩（信号，不是装饰）

- 90% 黑白灰
- 全站主色（Accent）仅选一种（两者择一）：
  - 国际橙 `#FF3B30`
  - 电光蓝 `#2997FF`
- 语义色（允许使用，但必须克制）：
  - 共识/生机：`#10B981`
  - 冲突警告：`#E11D48`
  - 中性：`#71717A`

### 7.5 组件语言

- 卡片：直角或极小圆角（2px）
- 边框：1px 实线边框（默认低对比；hover 提亮）
- 布局：杂志化
  - 左侧（或上方）大号数据（如 85%）
  - 右侧（或下方）摘要
- 输入框：仅底边框，无背景
- Tab：纯文字；激活态用字体加粗或 1px 点标示

### 7.6 动效与反馈（微观精密）

- Hover：瞬间响应（边框提亮），禁止慢吞吞渐变
- 点击反馈：机械干脆
- 转场：短（建议 150ms），禁止拖泥带水
- 无障碍：尊重 `prefers-reduced-motion`

### 7.7 UI 验收清单（给 AI 自检）

- 所有页面元素是否严格对齐网格？
- 是否存在任何“装饰性”元素？若存在必须删除
- 是否使用了超过 1 个主色？若是必须合并
- 是否出现渐变字/发光按钮/胶囊按钮？若是必须重做

---

## 8. 内容与合规（MVP 最低要求）

- 面向中国市场：
  - 文案必须自然中文
  - 避免政治敏感内容的引导
- AI 输出需提示不确定性与免责声明（克制、不过度）
- 需要提供一个“举报/反馈”入口（MVP 可简单）

---

## 9. 质量门槛（Definition of Done）

- 核心旅程 A/B/C 全部可走通
- 任何一次对话可以导出：
  - 结构化观点/分歧/共识
  - 沟通策略/话术建议
- UI 满足 7 章设计规范，且在移动端可用
- 无明显阻塞性 bug（崩溃、白屏、发送失败、会话丢失）

## 10. 待确认问题（请你回复，我会把答案写回本文档）

1. **登录/匿名**：MVP 是否必须支持登录？还是允许匿名游客使用（会话是否持久化）？
2. **会话持久化范围**：
   - 是否需要“历史会话列表”？
   - 历史保留多久？
3. **Agent 上限**：单个会话最多允许选择几个 Agent（例如 3/5/7/10）？
4. **主色选择**：全站 Accent 选 `#FF3B30`（国际橙）还是 `#2997FF`（电光蓝）？
5. **内容边界**：是否需要在 MVP 内做敏感词/风险内容拦截？还是先以提示为主？
