1) 市场与切入（TAM/SAM/SOM 的可证据化）
[目标用户的“最先买单者”到底是谁]
你现在覆盖创作者/职场决策者/学生/普通用户。你判断 MVP 的 1st ICP 是哪一类？为什么不是另外两类？
[需求强度的量化指标]
你怎么证明“多视角理解”不是 nice-to-have？有没有可以量化的代理指标（例如：每周主动发起话题次数、为同一话题付费复盘的意愿、创作者复用率）？
[切入的垂直话题域]
初期你说聚焦“职场、生活决策”。更具体：前 3 个一级类目是什么？每个类目的供给、敏感度、可传播性如何评估？
[分发场景与渠道假设]
你期待的主分发是站内信息流增长，还是小红书/抖音/朋友圈外部分享带回流？如果主增长靠外部分享，你的内容形态与平台规则（限流/敏感/引流）如何匹配？
2) 产品与留存（从“看卡片”到“深度互动”的真实漏斗）
[关键漏斗的基线假设]
“刷卡片 -> 进圆桌 -> 生成产出 -> 分享/付费”各环节你期望的转化率分别是多少？依据是什么？
[留存机制是什么，不是愿景]
用户第二天/第七天为什么回来？是关注话题更新、关注某个“专家人设”、还是“我的私人圆桌进度/交付物迭代”？
[UGC 还是 PGC 还是 AIGC 的供给结构]
热门话题是系统预设为主还是用户创建为主？如果用户创建占比上升，你如何避免质量塌方与敏感风险？
[“理解地图”的产品价值验证]
你如何证明理解地图指标能提高用户满意度/停留/分享/付费，而不是一个“好看的仪表盘”？
3) 内容质量与差异化（你最大的风险点）
[你如何定义“有趣/有深度”的客观标准]
你会用哪些可度量指标评估内容质量：完读率、二跳率、收藏率、分享率、评论/追问率、二次创作率？
[专家人设的规模化方法]
“专家人设打磨”短期有效，但扩到 100/1000 个角色怎么办？是模板化、程序化生成、还是少量精品角色覆盖大多数场景？
[避免“稻草人”与立场公平的机制]
你怎么检测某一方观点被弱化/被 caricature？有没有自动化的“论证强度”或“对等钢人化”检查？
[事实核查的成本与责任边界]
求真者要做 deep research 并标注来源：
数据源来自哪里？
错误引用/过期信息造成误导时，责任如何界定？
有没有“可回溯证据包”的产品形态？
4) 技术路线可行性（AIGC 生成代码/渲染管线的工程风险）
[你现在的“卡片生成”技术现状]
现在离你愿景（生成 React/Tailwind/SVG + 流式渲染）差距在哪？是模型能力、Prompt、AST 解析器、运行时沙箱、安全、还是设计语言不成熟？
[最难的 2 个技术风险是什么]
我更关心你认为最难且最可能卡住的点，以及你的 fallback 方案（例如：从“生成代码”退回“生成结构化 DSL + 渲染器”）。
[内容安全与前端代码安全]
如果 AI 生成可执行前端代码：如何防止注入、访问全局对象、盗取 token、发起恶意请求？渲染环境是否隔离？
[成本结构与延迟体验]
一次“公开话题卡片生成”平均消耗多少 tokens/多少钱/多长时间？
一次“私人圆桌深度讨论 + 交付物”成本？
在增长后你靠什么降本：缓存、复用、分层模型、异步生成？
5) 商业模式与定价（愿付费的人是谁、为什么现在付）
[付费触发点设计]

[单次深度报告 ¥9.9 的内容边界]

[创作者订阅 ¥99/月 的 ROI 证明]

[B 端 API 的真实场景]
