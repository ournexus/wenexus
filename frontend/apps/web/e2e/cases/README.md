# 语义化测试用例

本目录存放业务级语义测试用例，描述"**测什么**"而不是"**怎么操作**"。

## 设计原则

- **不绑定 DOM 结构**：用例只描述业务意图和验证点，不包含 CSS 选择器
- **双模式可执行**：
  - 人工/AI 执行：根据语义描述在浏览器中操作并验证
  - 自动化执行：未来 UI 稳定后，可基于用例编写 Playwright 实现层
- **用例即规格书**：每个用例文件是对应域的行为规格

## 用例格式

```yaml
domain: 域名称
feature: 功能名称
cases:
  - id: 唯一标识
    name: 用例名称
    preconditions: 前置条件列表
    steps: 操作步骤（自然语言）
    expected: 预期结果（验证点）
    priority: P0 | P1 | P2
```

## 执行方式

### AI 执行（推荐开发阶段使用）

让模型读取 YAML 用例，通过浏览器（Playwright / Chrome DevTools）按步骤操作并验证。
模型可以自己找到页面元素，不需要硬编码选择器。

### 自动化执行

待 UI 稳定后，根据语义用例编写对应的 Playwright spec 文件放入 `e2e/health/` 或新建实现目录。
